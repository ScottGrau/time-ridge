Open ID Connect (OIDC) Overview

What is OIDC?

OIDC is a highly secure, loosely coupled token based application level protocol typically used to authenticate users and serices. Often called a "flow" the authentication process usually requires multiple redirects and requests over https resulting in a JWT token that can be used by both web client and api services as different areas of a RIA or web site are accessed.

How is OIDC Used?

  • The actual exchange of user credentials (username/password, retinal scan, etc) occurs within the IdP domain when configured
  • Also can be used to securely authenticate services using using private key (client secret) instead of password
  • The OIDC protocol allows the "flow" or protocol configuration to be altered based on environment application need, which are called Grant Types and desribed below

Grant Types

OIDC is decoupled and can run on many platforms and environments requiring the protocol to be highly configurable based on application need. The configuration consists of channging Grant Type, which will alter the http sequence ("flow") required by the protocol to receive a JWT token.
